These are YAMP's own reference screenshots (jianyu-li/yet-another-media-player), grouped by the template we'd set on Music-Alt1. In our setup each maps to a custom:ted-music-card with engine: yamp — the wrapper resolves this device's Music Assistant player and feeds it to YAMP. Click any image to open full-size.

large_modern recommended

Big cover‑art background, modern (more‑info style) controls, adaptive control sizing, dynamic card height. Best fit for a full kiosk content area.

largepreview.png
modern_controls_preview.png
Music-Alt1 config for this look 
type: custom:ted-music-card
player_source: settings
engine: yamp
fill: true
yamp_config:
  template: large_modern
  idle_screen: search        # now-playing when playing, search when idle
  match_theme: true
  artwork_object_fit: cover

huge_yamp

Maximized controls, large text, and a giant progress bar for across‑the‑room viewing — essentially a scaled‑up large_modern. No dedicated screenshot in the repo; the large_modern shots are the closest reference.

  match_theme: true

minimal_mini

Compact mini card with no visible artwork, stepper volume, clean text alignment. Better suited to small tiles than a full‑page view.

minimal-preview.png

normal_mini

Standard compact mini player with a blurred background‑artwork effect (shown here collapsed).

collapsed.png

quick_and_easy

High‑accessibility layout: persistent entity chip row, volume overlays, and always‑active quick‑grouping. No dedicated screenshot; the multi‑layout overview shows the chip‑row style.

layouts.png (overview)
Note: with “music player only”, the chip row is hidden by default (show_chip_row: auto), so quick_and_easy's grouping benefits mostly apply if we later expose multiple Music Assistant players.
